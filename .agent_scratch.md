# Analysis of Connection Logic

## Definitions
- **Adjacent**: Two tiles that share a side (up, down, left, right). In grid terms, |r1 - r2| + |c1 - c2| = 1.
- **Touching/靠着**: Same as adjacent in this context. The issue is that the current code draws a connection line even for directly adjacent tiles that can be matched without any path (since they are next to each other).

## Current Behavior
The `getConnectionPath` function returns a direct path for horizontally or vertically adjacent tiles:
- For horizontal adjacency (same row, columns differ by 1): returns [center1, center2]
- For vertical adjacency (same column, rows differ by 1): returns [center1, center2]

This causes `drawConnectionPath` to draw a line between them, which is unnecessary since they are already touching.

## Required Change
Modify `getConnectionPath` to return `null` for directly adjacent tiles (where the distance is exactly 1), so no line is drawn. Only draw lines for tiles that require a path (distance > 1 or with turns).

## Implementation Plan
1. In `getConnectionPath`, after checking for same cell, add a check for direct adjacency (Manhattan distance = 1).
2. If adjacent, return `null` to skip drawing the connection line.
3. Keep all other path logic unchanged.
